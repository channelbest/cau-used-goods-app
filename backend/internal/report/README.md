# 举报模块 (Report)

## 模块职责

举报模块负责处理用户对商品、用户、订单的举报功能，包括提交举报、查看举报状态、管理员处理举报等。

## 关联数据表

- `reports` — 举报主表
- `report_images` — 举报凭证图片表

## 核心规则

1. 举报类型：`PRODUCT`（商品）、`USER`（用户）、`ORDER`（订单）
2. 举报状态：`PENDING`（待处理）、`PROCESSING`（处理中）、`RESOLVED`（已解决）、`REJECTED`（已驳回）、`CLOSED`（已关闭）
3. 同一用户对同一目标只能有一个待处理的举报
4. 举报可以附带多张凭证图片

## 主要接口

### 用户端

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | `/reports` | 提交举报 | 登录+认证 |
| GET | `/reports/my` | 我的举报列表 | 登录+认证 |
| GET | `/reports/:id` | 举报详情 | 登录+认证 |

### 管理员端

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/admin/reports` | 举报列表（管理员） | 登录+管理员 |
| POST | `/admin/reports/:id/handle` | 处理举报 | 登录+管理员 |

## 权限校验

- 用户端接口需要 JWT 登录 + 学生认证
- 管理员端接口需要 JWT 登录 + `role=ADMIN`

## 异常情况

| 场景 | 错误信息 |
|------|---------|
| 重复举报同一目标 | "you have already reported this target" |
| 非管理员访问管理接口 | 403 Forbidden |

## 与概要设计一致的地方

- 举报类型和状态定义一致
- 管理员权限校验一致
- 图片单独建表一致

## 仍需讨论的问题

- 举报处理后是否需要通知举报人？
- 举报成立后的后续操作（如下架商品、禁用用户）是否在本模块实现？
