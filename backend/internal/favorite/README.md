# 收藏模块 (Favorite)

## 模块职责

收藏模块负责处理用户对商品的收藏功能，包括收藏、取消收藏、查询收藏状态等操作。

## 关联数据表

- `favorites` — 收藏表
- `products` — 商品表（关联查询商品信息）

## 核心规则

1. 同一用户对同一商品只能收藏一次（唯一约束）
2. 取消收藏是软删除（`is_deleted = 1`），不是物理删除
3. 再次收藏同一商品会恢复之前的记录

## 主要接口

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | `/favorites` | 收藏商品 | 登录+认证 |
| GET | `/favorites` | 收藏列表 | 登录+认证 |
| GET | `/favorites/check` | 检查是否收藏 | 登录+认证 |
| DELETE | `/favorites/:productId` | 取消收藏 | 登录+认证 |

## 权限校验

- 所有接口需要 JWT 登录
- 所有接口需要学生认证（`auth_status=VERIFIED`）

## 异常情况

| 场景 | 错误信息 |
|------|---------|
| 商品不存在 | 收藏失败（外键约束）|

## 与概要设计一致的地方

- 软删除设计一致
- 接口设计一致
