# 前端成员 A 页面设计

## 负责范围

成员 A 负责以下页面：

1. 登录页
2. 学生认证页
3. 首页
4. 搜索结果页
5. 分类商品页
6. 商品详情页

## 1. 登录页

页面路径：

/pages/login/login

页面作用：

用户进入小程序后，先进行微信登录。

页面内容：

- 小程序名称
- 登录按钮
- 登录失败提示

按钮：

- 微信登录按钮
  - 点击后调用登录接口
  - 登录成功后跳转到学生认证页或首页

跳转：

- 未认证用户：跳转学生认证页
- 已认证用户：跳转首页

需要接口：

- 登录接口
- 获取当前用户信息接口

## 2. 学生认证页

页面路径：

/pages/auth/student-auth

页面作用：

让用户提交学生身份认证信息。

页面字段：

- 姓名
- 学号
- 学院
- 学生证图片

按钮：

- 上传学生证图片
- 提交认证

跳转：

- 认证成功后跳转首页
- 认证失败时停留当前页并提示错误信息

需要接口：

- 提交学生认证接口
- 图片上传接口

## 3. 首页

页面路径：

/pages/home/home

页面作用：

展示商品列表，提供搜索和分类入口。

页面内容：

- 搜索框
- 分类入口
- 商品列表
- 商品图片
- 商品标题
- 商品价格
- 商品成色
- 发布时间

按钮/操作：

- 点击搜索框：进入搜索结果页
- 点击分类：进入分类商品页
- 点击商品：进入商品详情页

需要接口：

- 首页商品列表接口
- 分类列表接口

## 4. 搜索结果页

页面路径：

/pages/search/search

页面作用：

展示用户搜索到的商品。

页面字段：

- 搜索关键词
- 商品列表
- 筛选条件

筛选条件：

- 分类
- 价格区间
- 新旧程度
- 发布时间

按钮/操作：

- 点击商品：进入商品详情页
- 修改筛选条件：重新请求商品列表

需要接口：

- 搜索商品接口

## 5. 分类商品页

页面路径：

/pages/category/category

页面作用：

展示某个分类下的商品。

页面内容：

- 当前分类名称
- 商品列表
- 排序方式

排序方式：

- 最新发布
- 价格从低到高
- 价格从高到低

按钮/操作：

- 点击商品：进入商品详情页

需要接口：

- 分类商品列表接口

## 6. 商品详情页

页面路径：

/pages/goods/detail

页面作用：

展示商品详细信息。

页面内容：

- 商品图片
- 商品标题
- 商品价格
- 商品描述
- 商品成色
- 商品分类
- 卖家信息
- 发布时间

按钮：

- 收藏
- 提交预约
- 举报

权限规则：

- 未登录用户点击收藏、预约、举报时，提示先登录
- 已登录但未学生认证时，提示先完成学生认证
- 已认证用户可以正常操作
- 商品详情页不直接展示卖家明文联系方式

跳转：

- 点击提交预约：跳转提交预约页
- 点击举报：跳转举报页

需要接口：

- 商品详情接口
- 收藏接口
